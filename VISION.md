# VISION.md — wb-seller-agent: продуктовое видение

## Суть продукта

AI-финансист и операционный менеджер для WB-продавца. Заменяет дорогого и неточного финансового директора, даёт реальный P&L с интерпретацией и рекомендациями — не просто цифры, а "вот где ты теряешь деньги и что делать".

Продавец управляет бизнесом через единый интерфейс — не разбираясь в технологиях.

## Инсайты из кастдевов (март 2026, 5 интервью)

**Главная боль (4/5):** финансовая отчётность — ДДС, PnL, платёжный календарь. Ведут вручную или платят дорого за фин. директора с плохим качеством.

**Цитата-триггер (Влад Кубатин):**
> "Бьётся моя финансовая отчётность просто потому что задорого, много денег плачу, нету точности данных, никто не перепроверяет. Я хочу финансового менеджера взять — вот это меня бесит."

**Конкуренты которые уже есть:** 10x/X10, TrueStats, Evirma, Neuron Kook (отзывы), ChatGPT.

**Что НЕ платят:** ещё один дашборд с теми же цифрами, дублирование функций 10x.

**WOW-момент:** не "посмотри на данные", а "вот где ты теряешь деньги и почему — конкретно".

**Ценовой диапазон:** 3 000–10 000 ₽/мес (все 5 готовы платить).

## Эволюция продукта

### MVP — Telegram-бот (сейчас)

**Что умеет:**
- Ответить на отзывы и вопросы покупателей
- Показать P&L с реальными комиссиями WB
- Показать баланс и статистику заказов

**WOW-момент MVP:**
```
Продавец: "Как у меня дела в марте?"

Агент:
— Выручка: 847 000 ₽
— Реальная прибыль: 475 000 ₽ (56%)
— ⚠️ Товар #12345 убыточен — возвраты 34%,
  логистика съедает маржу
— ⚠️ Неликвид на 120 000 ₽ — рекомендую
  снизить цену на 15% для ускорения оборота
```

Это то чего не даёт ни MPStats, ни 10x, ни фин. директор.

**Почему Telegram:** продавцы уже там, нет порога входа, быстро запустить, быстро проверить спрос.

**Ограничение:** таблицы и аналитика читаются плохо → решается в v2.

---

### v2 — Telegram + Web Dashboard (после 10 платящих клиентов)

**Что добавляется:**
- Простой веб-интерфейс для отчётов и аналитики
- 2-3 агента: Reviews Agent, Finance Agent, первый Analytics Agent
- Видимость работы агентов: что делает каждый, какие решения принял
- Telegram остаётся для уведомлений и быстрых действий

**Пример:**
```
Telegram: "⚠️ Упала конверсия по товару #12345. Открой дашборд."
Веб: график конверсии, причина, рекомендация агента
```

**Стек:** React + простой бэкенд, hosted SaaS (не self-hosted).

---

### v3 — Полная оркестрация агентов (после Product-Market Fit)

**Видение:** агенты работают как команда, продавец видит как они взаимодействуют.

**Пример взаимодействия агентов:**
```
Analytics Agent → Content Agent:
"По ключевому запросу 'футболка мужская' позиция упала с 12 до 34.
 Рекомендую добавить ключи в описание карточки #12345."

Ads Agent → Content Agent:
"Запускаю A/B тест для кампании #789.
 Нужны 2 варианта обложки с разным акцентом на цену/качество."

Content Agent → продавцу:
"Получил задачи от Analytics и Ads агентов.
 Подготовил изменения. Подтвердить?"
```

**Что видит продавец:**
- Лента активности агентов (что делают прямо сейчас)
- Граф взаимодействий (кто кому что передал)
- Лог решений с объяснениями
- Возможность вмешаться в любой момент

**RAG на курсах по продажам:**
- Каждый агент обучен на базе знаний из реальных курсов WB-продаж
- Content Agent знает принципы продающих карточек
- Ads Agent знает стратегии ставок и ДРР
- Analytics Agent знает как интерпретировать метрики WB

**Инспирация по UI:** Paperclip (github.com/paperclipai/paperclip) — org chart агентов, лента активности, бюджеты. Адаптировать под продавца: убрать dev-специфику, добавить WB-метрики.
